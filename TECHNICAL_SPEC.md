## Data Pipeline: How Scoring Data Is Sourced

**The `patents` table has no `abstract` column.** Abstracts are sourced from the PatentsView API (already integrated in patent-tracker) and stored in `patent_scores.abstract`.

**Scoring input per patent:**
- `title` — from D1 `patents` table
- `cpc_section` — from D1 `patents` table (passed to scorer as context, NOT used to hard-skip)
- `abstract` — fetched from PatentsView API (`patent_abstract` field)
- `has_diagrams` — checked via lightweight Google Patents scrape

**CPC as Ordering + Context (not a hard filter):**

CPC section is passed to Claude Haiku as context and used to order batches for a higher early hit rate — but never to hard-skip patents. Velcro is class D. Teflon is class C. Gore-Tex is class D. A blanket letter-level skip would silently eliminate iconic patents. Let the AI scorer decide.

```typescript
// Order batches to surface higher-probability candidates first
// A (Human Necessities) and H (Electronics) first, then everything else
// But ALL sections are eligible — no hard skips

const D1_QUERY = `
  SELECT patent_number, title, cpc_section, assignee_name,
         calculated_expiration_date, filing_date, grant_date
  FROM patents
  WHERE status = 'Expired'
    AND enriched = 1
    AND title IS NOT NULL
    AND patent_number NOT IN (SELECT patent_number FROM patent_scores)
  ORDER BY
    CASE WHEN cpc_section IN ('A', 'H') THEN 0 ELSE 1 END,
    RANDOM()
  LIMIT 50
`
```

**PatentsView API — Abstract Fetch:**

PatentsView returns `patent_abstract` for any US patent number. The API key is already in the codebase (`PATENTSVIEW_API_KEY`).

```typescript
async function fetchAbstract(patentNumber: string, apiKey: string): Promise<string | null> {
  const q = encodeURIComponent(JSON.stringify({ patent_id: patentNumber }))
  const url = `https://search.patentsview.org/api/v1/patent/?q=${q}&f=["patent_abstract"]`

  const response = await fetch(url, {
    headers: { 'X-Api-Key': apiKey }
  })
  if (!response.ok) return null
  const data = await response.json() as any
  return data.patents?.[0]?.patent_abstract ?? null
}
```

**Rate limit: 45 requests/minute.** Enforce this strictly. Use the implementation below for all PatentsView calls:

```typescript
// Shared rate limiter — use for ALL PatentsView calls
const patentsViewLimiter = {
  queue: [] as number[],
  async throttle(): Promise<void> {
    const now = Date.now()
    this.queue = this.queue.filter(t => now - t < 60000)
    if (this.queue.length >= 45) {
      const wait = 60000 - (now - this.queue[0]) + 50
      await new Promise(resolve => setTimeout(resolve, wait))
      return this.throttle()
    }
    this.queue.push(Date.now())
  }
}

// Usage: always call before any PatentsView request
await patentsViewLimiter.throttle()
const response = await fetch(url, { headers: { 'X-Api-Key': apiKey } })
```

**Why not ODP API?** The ODP API handles prosecution data (PTA, PTE, MF events, continuity). It does not return patent text. PatentsView is the correct source for abstract.

**Why not scrape Google Patents for abstract?** Scraping is fragile (HTML structure changes, rate limiting, bot detection). PatentsView is a stable official USPTO API. Use it.

---

**Content Generation — Text Sources:**

PatentsView secondary text endpoints (`g_brf_sum_text`, `g_draw_desc_text`) have inconsistent coverage — confirmed returning zero results on real patents. Do not rely on them.

For approved patents going into content generation, the content generator uses:
1. `abstract` — already stored in `patent_scores`, pulled forward (no extra API call)
2. Google Patents description scrape — first 3000 chars of the detailed description, extracted from the patent page (approved patents only, low volume, randomized timing)

Claude Sonnet receives both and translates dense technical language into consumer copy. The abstract provides the core invention summary; the description provides supporting details, real-world context, and the "surprising fact" angle that makes web content worth reading.

## Key Algorithms

### Deterministic URL Slug Generator

```typescript
/**
 * Generate a URL-safe slug from patent number and title
 * CRITICAL: Must be deterministic - same inputs always produce same output
 */
function generateSlug(patentNumber: string, title: string): string {
  // Extract meaningful keywords from title
  const keywords = title
    .toLowerCase()
    .replace(/[^a-z0-9\s-]/g, '')  // Remove special chars
    .split(/\s+/)                   // Split on whitespace
    .filter(word =>
      word.length > 3 &&            // Filter short words
      !STOP_WORDS.includes(word)    // Filter common words
    )
    .slice(0, 5)                    // Take first 5 meaningful words
    .join('-')

  // Combine patent number + keywords
  // Example: US6025810-vertical-mouse-wrist-health
  return `${patentNumber.toLowerCase()}-${keywords}`
}

const STOP_WORDS = ['the', 'and', 'for', 'with', 'method', 'system', 'apparatus', 'device']

/**
 * Generate full URL (always available, even before publishing)
 */
function generateURL(slug: string): string {
  return `https://YOURBRAND.com/patent/${slug}`
}
```

### Google Patents Diagram Checker

**IMPORTANT:** The patent number in D1 is stored as digits only (e.g., `7325258`).
Google Patents requires the `US` prefix: `https://patents.google.com/patent/US7325258`.
Always prepend `US` when building Google Patents URLs.

```typescript
/**
 * Quick scrape to check if diagrams exist (for scoring phase)
 * Used for 50 patents per batch scoring run — not a high-frequency operation
 */
async function checkForDiagrams(patentNumber: string): Promise<boolean> {
  // patentNumber is digits only from D1 — must add US prefix for Google Patents
  const url = `https://patents.google.com/patent/US${patentNumber}`
  const html = await fetch(url).then(r => r.text())

  // Check for diagram URLs in page source
  const hasImages = html.includes('patentimages.storage.googleapis.com')
  return hasImages
}

/**
 * Full scrape to get diagrams for content generation
 * Only called for approved patents (post-scoring)
 */
async function scrapePatentDiagrams(patentNumber: string): Promise<string[]> {
  const url = `https://patents.google.com/patent/US${patentNumber}`
  const html = await fetch(url).then(r => r.text())

  // Extract diagram URLs (get top 3)
  const imageRegex = /https:\/\/patentimages\.storage\.googleapis\.com\/[^"]+\.png/g
  const images = Array.from(html.matchAll(imageRegex))
    .map(match => match[0])
    .filter((url, index, self) => self.indexOf(url) === index)  // Unique only
    .slice(0, 3)  // Top 3

  return images
}

/**
 * NOTE: Google Patents is used for DIAGRAMS + DESCRIPTION during content generation.
 * PatentsView secondary text endpoints (g_brf_sum_text, g_draw_desc_text) have
 * inconsistent coverage and cannot be relied upon. Google Patents description scrape
 * is the fallback — approved patents only, low volume, randomized timing.
 */

/**
 * Full scrape for content generation — approved patents only
 * Returns both diagram URLs and first 3000 chars of description
 */
async function scrapePatentForContent(patentNumber: string): Promise<{
  images: string[]
  description: string
}> {
  // Random delay 1-3 seconds — cautious approach, not batched
  await new Promise(resolve => setTimeout(resolve, 1000 + Math.random() * 2000))

  const url = `https://patents.google.com/patent/US${patentNumber}`
  const html = await fetch(url, {
    headers: { 'User-Agent': 'Mozilla/5.0 (compatible; research-bot/1.0)' }
  }).then(r => r.text())

  // Extract diagram URLs (top 3)
  const imageRegex = /https:\/\/patentimages\.storage\.googleapis\.com\/[^"]+\.png/g
  const images = Array.from(html.matchAll(imageRegex))
    .map(match => match[0])
    .filter((url, index, self) => self.indexOf(url) === index)
    .slice(0, 3)

  // Extract description (first 3000 chars)
  const descMatch = html.match(/<section itemprop="description">(.+?)<\/section>/s)
  const description = descMatch ? stripHtml(descMatch[1]).substring(0, 3000) : ''

  return { images, description }
}

function stripHtml(html: string): string {
  return html
    .replace(/<[^>]*>/g, '')
    .replace(/&nbsp;/g, ' ')
    .replace(/&amp;/g, '&')
    .replace(/\s+/g, ' ')
    .trim()
}
```

### PatentsView API — Abstract Fetching

PatentsView is the canonical source for patent abstracts. Already used in patent-tracker; API key is in `.env.local` as `PATENTSVIEW_API_KEY`.

```typescript
async function fetchPatentAbstract(
  patentNumber: string,
  apiKey: string
): Promise<string | null> {
  await patentsViewLimiter.throttle()  // Always throttle before PatentsView calls
  const q = encodeURIComponent(JSON.stringify({ patent_id: patentNumber }))
  const url = `https://search.patentsview.org/api/v1/patent/?q=${q}&f=${encodeURIComponent(JSON.stringify(['patent_abstract', 'patent_id']))}`

  const response = await fetch(url, {
    headers: { 'X-Api-Key': apiKey }
  })
  if (!response.ok) return null
  const data = await response.json() as any
  if (data.error || !data.patents?.length) return null
  return data.patents[0].patent_abstract ?? null
}
```

Rate limit: 45 requests/minute. Process in batches of 10 with `Promise.all`, add 700ms delay between batches.

**For Next.js on Vercel:** Use the D1 REST API. `getRequestContext()` from `@cloudflare/next-on-pages` only works on Cloudflare Pages deployments — NOT Vercel. This project deploys to Vercel, same as patent-tracker.

// lib/db.ts — reuse this pattern from patent-tracker
const CLOUDFLARE_ACCOUNT_ID = process.env.CLOUDFLARE_ACCOUNT_ID!
const CF_D1_DATABASE_ID = process.env.CF_D1_DATABASE_ID!
const CLOUDFLARE_API_TOKEN = process.env.CLOUDFLARE_API_TOKEN!

async function queryD1(sql: string, params: any[] = []) {
  const response = await fetch(
    `https://api.cloudflare.com/client/v4/accounts/${CLOUDFLARE_ACCOUNT_ID}/d1/database/${CF_D1_DATABASE_ID}/query`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${CLOUDFLARE_API_TOKEN}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ sql, params }),
    }
  )
  const data = await response.json() as any
  return data.result[0].results
}

// Example: fetch unscored expired patents (CPC ordering only, no hard skips)

## Error Handling

### Worker Error Responses
```typescript
// Always return structured errors
try {
  const result = await scorePatents()
  return Response.json({ success: true, data: result })
} catch (error) {
  console.error('Scoring error:', error)
  return Response.json(
    { success: false, error: (error as Error).message },
    { status: 500 }
  )
}
```

### Admin UI Error States
```typescript
// Show user-friendly messages
try {
  await approvePatent(id)
  toast.success('Patent approved for content generation')
} catch (error) {
  toast.error('Failed to approve patent. Please try again.')
  console.error(error)
}
```

## Deployment Steps

**Order matters. Do not skip ahead.**

1. **Create D1 tables first:**
   ```bash
   wrangler d1 execute patent-tracker-db --file=schema.sql
   ```

2. **Create Cloudflare R2 bucket:**
   ```bash
   wrangler r2 bucket create patent-images
   ```
   Then configure for public access in the Cloudflare dashboard.

3. **Set Worker secrets:**
   ```bash
   wrangler secret put ANTHROPIC_API_KEY && wrangler secret put PATENTSVIEW_API_KEY
   ```

4. **Deploy workers:**
   ```bash
   cd workers && wrangler deploy
   ```
   Copy worker URLs into Next.js env vars.

5. **Deploy Next.js to Vercel:**
   Add env vars in Vercel dashboard, then `git push` to trigger auto-deploy.

6. **Test end-to-end workflow**
